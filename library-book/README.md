## How to Use This Book

**Evaluating this library?** Read Chapters 0-2. That tells you what it does, why, and how it thinks.

**Just want to use it?** Chapter 3 gets you running, Chapter 5 covers the main features.

**Hitting weird issues?** Chapter 7 is your friend.

**Want to contribute?** Chapter 8 has the dev setup, then read Chapter 4 for internals.

---

## Keeping This Book Alive

Update it when:

- You add or change public API (Chapters 5, 10)
- You change architecture or internals (Chapter 4)
- You find a common user mistake (Chapter 7)
- You change the dev/build process (Chapter 8)

If a chapter doesn't apply to your project, delete it. A shorter book that's accurate beats a complete one that's stale.

| If your project... | Cut these chapters |
|--------------------|--------------------|
| Has no advanced/power-user features | Chapter 6 (Advanced Usage) |
| Is a personal project, not open source | Chapter 8 (Contributing) |
| Is a simple utility with minimal API | Appendix API section |
| Has no external dependencies or config | Appendix Configuration/Environment sections |
